## Phase 0: Automator Engine ✅ DONE

> UI 自动化核心引擎

- UiObject 封装 (AccessibilityNodeInfo 包装)
- Filter 过滤器 (文本、ID、类名、描述、bounds、clickable 等)
- UiGlobalSelector + SelectorParser (JSON 选择器解析)
- SearchAlgorithm (DFS / BFS)
- GlobalActionAutomator (dispatchGesture + performGlobalAction)
- 单元测试 62 tests 全覆盖

---

## Phase 1: Kotlin Adapter Layer ✅ DONE

> 将 Android API 封装为统一的 Adapter 接口

- AutomatorAdapter: findOne/find, click/longClick/setText/scroll, clickPoint/swipe/gesture, back/home/recents
- AppAdapter: launchApp, toast, clipboard, getAppName, isAppInstalled
- DeviceAdapter: getDeviceInfo, screenSize, battery, vibrate, brightness
- ShellAdapter: exec, execRoot, isRootAvailable
- EventAdapter: AccessibilityEvent/KeyEvent 转发
- StorageAdapter: SharedPreferences 封装
- UIAdapter: 悬浮窗管理，对话框显示

---

## Phase 2: HTTP REST API ✅ DONE

> 通过 HTTP API 对外暴露自动化能力

- HttpServer 基础架构 (原生 ServerSocket, Coroutine 异步, Express-like 路由)
- Middleware 系统 (CORS + Logger)
- AdapterContainer + ApiRoutes (Hilt DI 聚合, 统一路由注册)
- Controllers: Action, Ui, App, Device, Shell, Event, Screenshot, File, Log, Status, Static
- ResponseHelper (sendJson, sendError, sendBytes)

---

## Phase 3: Web Frontend 🔧 IN PROGRESS

> React Web 前端，替代 Compose 原生 UI

### ✅ Task 3.1 — 前端脚手架 + 静态资源服务
- Vite + React 19 + TypeScript
- StaticController: assets/web/ 静态文件服务 + SPA fallback
- api.ts: HTTP API 客户端封装

### ✅ Task 3.2 — Dashboard 页面
- 服务状态、设备信息展示

### ✅ Task 3.3 — UiInspector 页面
- 无障碍树 dump + 截图预览

### ✅ Task 3.4 — Controls 页面
- 坐标点击、滑动、按键、应用启动

### ✅ Task 3.5 — Shell 终端
- xterm.js 终端，Shell 命令执行

### Task 3.6 — UiInspector 增强
- **目标**: 截图上元素高亮，点击截图选中元素，属性面板
- **验收**: 可视化选择 UI 元素，查看属性，生成选择器

### Task 3.7 — 设置页面
- **目标**: HTTP Server 端口配置、权限状态检查
- **验收**: 修改配置后生效

### Task 3.8 — Compose UI 清理
- **目标**: 移除 `app/src/main/kotlin/com/autodroid/ui/` 下的 Compose 页面
- **保留**: ConsoleRepository (供 LoggerMiddleware 使用)
- **验收**: 删除无用 Compose 依赖，APK 体积减小

## Phase 4: API 完善 📋 PLANNED

> 补全 API 覆盖度，增强实时通信

### Task 4.1 — API 查漏补缺
- **目标**: 对照 Adapter 能力，补全缺失的 HTTP 端点
- **候选**: clipboard, getAppName, isAppInstalled, vibrate, brightness, 定时任务
- **验收**: 所有 Adapter 方法都有对应 HTTP 端点

### Task 4.2 — WebSocket 实时通信
- **目标**: 替代轮询，实时推送日志、事件
- **验收**: Web 前端通过 WebSocket 接收实时更新

## Phase 5: Advanced Features 📋 PLANNED

> 截图增强、OCR、找图

### Task 5.4 — PermissionAdapter & ScheduleAdapter
